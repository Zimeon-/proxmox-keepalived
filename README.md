# proxmox-keepalived
Proxmox Virtual IP for cluster management

The shell script will install keepalived and nginx to enable easier cluster management with a standard port on 443 using nginx and keepalived to manage a virtual ip for cluster management using unicast between keepalived members. By default nginx will use promox ssl certificates with symlinks.

Usage: 
1. Download install-keepalived.sh
2. chmod u+x ./install-keepalived.sh
3. ./install-keepalived.sh [options]


Options:
  --vip <ip>           Virtual IP address (e.g., 192.168.1.100)
  --interface <iface>  Network interface (e.g., vmbr0)
  --router-id <id>     Keepalived router ID (default: 51)
  --auth-pass <pass>   Authentication password for Keepalived (required for clusters)
  --auto-detect        Enable auto-detection of interface, IPs, and peers
  --verbose            Enable verbose output
  --force              Force overwrite existing configs
  --cleanup           Remove existing Keepalived + Nginx installation
  --help, -h           Show this help message

Examples:
  # First node
 ./install-keepalived.sh --vip 192.168.1.100 --auto-detect --auth-pass mypassword

  # Subsequent nodes
  ./install-keepalived.sh --vip 192.168.1.100 --auto-detect --auth-pass mypassword

  # Cleanup existing installation
  ./install-keepalived.sh --cleanup


Nginx SSL Certificate logic:
1. Use custom uploaded certificates if there are any (pveproxy-ssl.pem and pveproxy-ssl.ke)
2. Use Proxmox autogenerated self-signed certificates (pve-ssl.pem and pve-ssl.key)

Keepalived configuration notes:
1. No master is specified, let keepalived select the master
2. Use Unicast for keepalived communication
3. Automatically select peers from proxmox cluster nodes
